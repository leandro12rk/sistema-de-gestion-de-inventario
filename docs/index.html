Arquitectura del sistema

El sistema se organiza por microservicios y usa un API Gateway como punto de entrada para exponer una sola ruta base.

Product Service Catalogo, categorias y proveedores en puerto 9090.
Inventory Service Movimientos, stock y recepciones en puerto 9091.
Purchase Service Ordenes y detalles de compra en puerto 9092.
API Gateway Punto unico de entrada en puerto 9000 con /api/v01.
  • Product Service en puerto 9090.
  • Inventory Service en puerto 9091.
  • Purchase Service en puerto 9092.
  • Gateway en puerto 9000 con la ruta base /api/v01.
